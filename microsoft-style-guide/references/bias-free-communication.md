# Bias-free communication

Source: [Bias-free communication](https://learn.microsoft.com/en-us/style-guide/bias-free-communication)

## Core principle

All communication must be inclusive and diverse. Microsoft technology reaches every part of the globe.

## Gender-neutral language

Use gender-neutral alternatives for common terms:

| Use this | Not this |
|----------|----------|
| chair, moderator | chairman |
| humanity, people, humankind | man, mankind |
| operates, staffs | mans |
| sales representative | salesman |
| synthetic, manufactured | manmade |
| workforce, staff, personnel | manpower |

## Pronouns

- Don't use *he, him, his, she, her, hers* in generic references.
- Instead: rewrite with *you* (second person), use plurals, use *the/a* instead of a pronoun, refer to the person's role, or use *person/individual*.
- It's OK to use singular *they/their/them* for generic references.
- Don't use *he/she* or *s/he* constructions.
- When writing about a real person, use their preferred pronouns.

### Examples

| Use this | Not this |
|----------|----------|
| If you have the appropriate rights, you can set other users' passwords. | If the user has the appropriate rights, he can set other users' passwords. |
| Developers need access to servers in their development environments. | A developer needs access to servers in his development environment. |
| When the author opens the document... | When the author opens her document... |

## Diversity in scenarios

- In fictitious scenarios, use diverse names reflecting various gender identities and cultural backgrounds.
- Avoid stereotypes in job roles.
- Represent diverse perspectives in text and images: gender identity, race, culture, ability, age, sexual orientation, socioeconomic class.
- Show people in a variety of professions, settings, and locales.
- Avoid examples reflecting primarily Western or affluent lifestyles.
- Be inclusive of nontraditional family structures.

## Avoiding bias

- Don't make generalizations about people, countries, regions, or cultures — even positive ones.
- Don't use slang, especially if it could be cultural appropriation (e.g., *spirit animal*).
- Don't use profane or derogatory terms.
- Don't use terms with unconscious racial bias or military associations.

| Use this | Not this |
|----------|----------|
| primary/subordinate | master/slave |
| perimeter network | demilitarized zone (DMZ) |
| stop responding | hang |
| allowlist/denylist | whitelist/blacklist |

## Disability and accessibility

- Focus on people, not disabilities.
- Say "readers who are blind" or "customers with limited dexterity."
- Don't use words implying pity (*stricken with*, *suffering from*).
- Don't mention disability unless relevant to the content.

## Race and ethnicity

- Use title-style capitalization for: Asian, Black and African American, Hispanic and Latinx, Native American, Alaska Native, Native Hawaiian, Pacific Islander, Indigenous Peoples.
- Lowercase: multiracial, white.

## Geographic sensitivity

- If naming cities, countries, or regions, verify they aren't politically disputed.
- Use equivalent references — don't mix countries with states or continents.
- Be mindful of regional and political contexts.

## Militaristic language

Source: [Militaristic language](https://learn.microsoft.com/en-us/style-guide/militaristic-language)

Avoid terms associated with violence and military actions unless referring to physical combat operations.

### Cybersecurity replacements

| Use this | Not this |
|----------|----------|
| address; protect against; respond to | combat; fight; eliminate |
| cyberattack chain | (cyber) kill chain |
| cyberattacker; bad actor; threat actor | attacker; adversary |
| impact | blast radius |
| multilayered approach; defense-in-depth cybersecurity | defense-in-depth approach |
| protect; safeguard; defend | guard; ward |
| secured | locked down |
| security; protection; defense | fortifications; first line of defense; frontlines |
| security teams; security analysts; defenders | frontline analysts |
| vulnerabilities; points of access; external exposure | external attack surface |
| block list | blacklist |
| allow list | whitelist |

### Contextual exceptions

- **Attack**: OK with qualifying context ("malware attacks"). Without context, use *cyberattack* (one word, no hyphen).
- **Defend/defense**: OK if context makes cybersecurity reference clear.
- **Threat**: OK with qualifying context ("identity threat"). Without context, use *cyberthreat* (one word, no hyphen).
- **Threat intelligence**: OK in cybersecurity context. Don't shorten to "threat intel."

### Never use (in cybersecurity context)

air cover, bomb, email bomb, mail bomb, time bomb, enemy, enemies, enemy lines, go on the offensive, invade, invasion, missile, torpedo, nuke, go nuclear, strike, troops
